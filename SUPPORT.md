# Support

Thanks for using Cyborg7! Here's how to get help and where each kind of request goes.

## Read the docs first

Most questions are answered in the documentation. Start there:

- [Documentation](docs/README.md) — getting started, configuration, architecture, providers, Cybos, the CLI, self-hosting, and troubleshooting.

If you hit a problem, [Troubleshooting](docs/troubleshooting.md) and the [FAQ](docs/faq.md) are the fastest paths to an answer.

## Ask a question or get help

For usage questions, ideas, and general discussion, use **GitHub Discussions**:

- [GitHub Discussions](https://github.com/Cyborg7-com/cyborg/discussions)

This is the best place to ask "how do I…", share what you're building, or talk through a setup with the community and maintainers. Search existing threads first — someone may have already covered it.

## Report a bug or request a feature

Found a reproducible bug or want to propose a feature? Open an issue using the templates:

- [Open an issue](https://github.com/Cyborg7-com/cyborg/issues/new/choose)

Please include your version or commit, the steps to reproduce, what you expected, and what actually happened. Clear repros get fixed faster.

## Report a security issue

**Do not open a public issue, discussion, or pull request for security vulnerabilities.** Report them privately as described in [SECURITY.md](SECURITY.md) — via a GitHub Security Advisory or the security contact listed there. We'll acknowledge and work with you on a fix and disclosure timeline.

## Contribute

Want to fix a bug or add a feature yourself? See [CONTRIBUTING.md](CONTRIBUTING.md) for how to build, test, and open a pull request. All contributions land in this public repo.
