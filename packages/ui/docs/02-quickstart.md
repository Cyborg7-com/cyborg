# Quickstart

## Prerequisites

- Node.js 22+
- pnpm 9+
- A running Cyborg7 daemon (see `packages/server/`)

## Install dependencies

From the monorepo root:

```bash
pnpm install
```

## Start the dev server

```bash
cd packages/ui
pnpm dev
```

The UI starts at `http://localhost:5173` by default.

## Connect to a daemon

The login page (`/login`) asks for a daemon URL and an auth token. For local development:

- **URL**: `ws://localhost:3000` (default daemon port)
- **Token**: Use the dev token generated by the daemon's `createDevToken()` helper

Once authenticated, the UI fetches your workspaces and redirects to the workspace selector.

## What happens under the hood

```
Browser
  │
  ├─ /login                    Enter daemon URL + token
  │    └─ connectToServer()    Opens WebSocket, sends hello, authenticates
  │
  ├─ /workspace                Lists workspaces returned by auth
  │    └─ selectWorkspace()    Fetches channels, tasks, members
  │
  └─ /workspace/:id/channel/:channelId
       └─ selectChannel()      Fetches message history (paginated)
       └─ Real-time messages   Arrive via WebSocket events
```

1. `connectToServer(url, token)` opens a WebSocket to the daemon and sends a `hello` frame
2. `authenticate()` sends the JWT token and receives the user profile + workspace list
3. `selectWorkspace(ws)` fetches channels, tasks, and members in parallel
4. `selectChannel(channelId)` loads the last 50 messages
5. New messages arrive via the `channel_message` event and append to the reactive message list

## Session persistence

After a successful connection, the session (URL + token) is saved to `localStorage`. On page reload, the UI restores the session automatically — no re-login required. Call `clearSavedSession()` to force a logout.

## Customize the shell

The default shell includes 8 rail items (Chat, Tasks, Audit, Memory, Skills, Logs, Feedback, Settings). You can customize it from any component or layout:

```typescript
import { shellConfig } from "$lib/core/plugin.svelte.js";

// Remove a rail item
shellConfig.removeRailItem("audit");

// Change the app name in the sidebar header
shellConfig.configure({ appName: "My App" });

// Add a custom rail item
shellConfig.addRailItem({
  id: "dashboard",
  label: "Dashboard",
  icon: `<svg>...</svg>`,
  path: "/dashboard",
  position: "nav",
  order: 6,
});
```
